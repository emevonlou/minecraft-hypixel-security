This documentation describes risks, checks, and best practices
focused on the player's local system security.

The approach is defensive (Blue Team) and strictly local.
There is no interaction with servers, exploits, or bypass techniques.


## Common threats in the Minecraft ecosystem

### Malicious mods
- Data collection without consent
- Bundled adware or spyware
- Updates from unofficial sources

### Third-party launchers
- Excessive privileges
- Persistence mechanisms
- Environment manipulation

### External tools
- Disguised keyloggers
- Process injection
- Suspicious background communication


## Core principles

- Least privilege
- Transparency
- Local verification
- User awareness


## Script goals

Scripts are designed to:
- alert
- inform
- support informed decisions

They do not act automatically.
