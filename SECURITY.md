# Security Policy

## Reporting a vulnerability

If you find a vulnerability or unexpected behavior, please open an Issue
describing the situation clearly and respectfully.

This project focuses on defensive and educational security.
Do not submit exploit code or attack tools.


## What counts as a vulnerability

- Script behavior that exposes user data
- Commands that unintentionally modify the system
- Logic errors that may mislead users


## What does NOT count

- Feature requests
- False positives in script output
- Personal disagreements with design choices


## Response expectations

This is an educational project.
Responses may take time, but all valid reports will be reviewed.
