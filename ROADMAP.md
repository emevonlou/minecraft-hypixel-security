# Project Roadmap – Minecraft Hypixel Security

This roadmap outlines the planned evolution of this project, with a focus on
defensive security, education, and ethical practices.

The goal is clarity, not feature bloat.

---

## Phase 1 – Foundation (Completed)

- Repository structure (Linux / Windows / multilingual docs)
- Basic system security checks (Linux)
- Minecraft mod awareness checks (Linux)
- Documentation in PT-BR and EN
- Ethical and defensive positioning

Status: completed

---

## Phase 2 – Windows parity

- Document Windows PowerShell scripts
- Improve Windows security checks
- Align Linux and Windows script behavior
- Add clear usage instructions for Windows users

Status: planned

---

## Phase 3 – Script improvements

- Reduce false positives
- Improve output readability
- Add optional verbose mode
- Add clear exit codes

Status: planned

---

## Phase 4 – Player education

- Add explanations of common risks for players
- Explain how malicious mods usually spread
- Explain what this project does NOT detect
- Add security hygiene checklist for gamers

Status: planned

---

## Phase 5 – Project maturity

- Add LICENSE
- Add CONTRIBUTING guidelines
- Add SECURITY.md
- Use GitHub Issues as a Blue Team checklist

Status: planned

---

## Non-goals

This project will not:
- bypass game protections
- analyze other players
- interact with servers
- provide cheating tools

---

## Philosophy

Security awareness > fear  
Defense > accusation  
Education > automation
