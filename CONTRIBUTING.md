# Contributing Guidelines

Thank you for your interest in improving this project.


## Principles

- Defensive security only
- Educational focus
- Respect game rules and platform terms
- No offensive or exploitative contributions


## Code style

- Keep scripts readable
- Comment important logic
- Avoid unnecessary complexity
- Prefer clarity over cleverness


## Commits

Use clear messages that explain intent.
Example:
"Improve process detection clarity"


## Pull requests

Explain:
- what changed
- why it changed
- what it improves
